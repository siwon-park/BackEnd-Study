# 26_메서드 시그니처

> Method Signature

## 1. 메서드 시그니처

> Java에서 메서드를 구별하는 기준

메서드 시그니처 = 메서드 이름 + 파라미터의 타입 + 파라미터의 순서

메서드 시그니처가 다르면 Java에서는 다른 메서드로 취급한다.

※ 반환 타입, 접근 제어자, 예외, 파라미터 이름은 메서드 시그니처에 포함되지 않는다.

### 1) 오버로딩 조건

오버로딩(Overloading); 흔히 메서드 오버로딩이라고도 하며, 과적(초과해서 싣다)의 의미로 동일한 이름의 메서드가 다수 존재하는 것을 의미한다.

즉, 오버로딩이 가능하다는 것은 동일한 이름의 메서드(실제로는 다른 메서드)가 여러 개 존재 가능하다는 의미와 같다.

#### (1) 오버로딩 가능 조건 (다른 메서드로 인식하는 조건)

동일한 이름의 메서드에서 다음 중 하나라도 다르면 다른 메서드로 취급한다.

- 파라미터의 개수
- 파라미터의 타입
- 파라미터의 순서

#### (2) 예시

```java
// 1. 파라미터의 개수가 다름
public void print(String s) {}
public void print(String s1, String s2) {}

// 2. 파라미터의 타입이 다름
public void print(String s) {}
public void print(int n) {}

// 3. 파라미터의 순서가 다름
public void print(String s, int n) {}
public void print(int n, String s) {}
```

#### (3) 메서드 오버로딩이 불가능할 경우

메서드 시그니처가 같아서 메서드 오버로딩이 불가능할 경우, 같은 메서드가 2개 이상 있어서 컴파일 에러가 발생한다.
