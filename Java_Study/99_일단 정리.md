## 1. ArrayStoreException: java.lang.Integer

`Arrays.fill()`메서드를 통해서 배열에 숫자를 초기화하려 했는데 다음과 같은 에러를 마주하였다.

`Exception in thread "main" java.lang.ArrayStoreException: java.lang.Integer`

Java Doc을 찾아보니 Arrays.fill 메서드에 들어가는 매개 변수는 `1차원의 배열`이다.

![image](https://user-images.githubusercontent.com/93081720/229662068-8c2ba238-7bd8-41d8-b30c-258b1451a950.png)

`2차원의 배열`에서 사용하려고 해서 에러가 났던 것이었다.

다음과 같은 방법으로 해결할 수 있다.

```java
for (int i = 0; i < N; i++) {
    Arrays.fill(arr[i], -1);
}

for (int[] i : arr) {
    Arrays.fill(i, -1);
}
```

## 2. isEmpty() vs. isBlank()

두 메서드 다 null을 검사하지는 않기 때문에 null 체크를 하지 않으면 NPE가 발생하므로 유의!

### 1) isEmpty()

길이(사이즈)가 0인지 검사하는 메서드. 따라서 공백 혹은 스페이스(탭 등)이 있을 경우 false를 반환하므로 유의.

컬렉션(List, Set, Map 등)에도 있어서 사용 가능한 메서드.

```java
String text = "";
text.isEmpty();   // true

String text2 = "   ";
text2.isEmpty();  // false
```

### 2) isBlank()

길이가 0이거나(비어 있거나) 오로지 공백, 스페이스, 탭 등으로 구성된 경우도 포함해서 체크하는 메서드. Java11+부터 사용 가능하며, 문자열(String) 타입에만 있는 메서드이기 때문에 컬렉션 타입에는 사용 불가함.

```java
String text = "";
text.isBlank();        // true

String text2 = "   ";
text2.isBlank();       // true

String text3 = "\n\t";
text3.isBlank();       // true

String text4 = " a ";
text4.isBlank();       // false
```
