## 2. 3단계 캐시 레이어

```
1단계: singletonObjects        → 완전히 초기화된 Bean
2단계: earlySingletonObjects   → 껍데기 인스턴스 (필드 미주입 상태)
3단계: singletonFactories      → Bean을 만들 수 있는 Factory
```



A -> B -> A 순환 참조 시나리오 (A의 필드에 B가 있고, B의 필드에 A가 있음); Spring 2.6 이전

```
1. A 생성 시작
2. new A() 로 껍데기 인스턴스 생성
3. A의 ObjectFactory를 3단계(singletonFactories)에 등록
4. A의 필드 B를 채우려고 B 생성 시작
5. new B() 로 껍데기 인스턴스 생성
6. B의 필드 A를 채우려고 A를 요청
7. 1단계(singletonObjects) 확인 → 없음
8. 2단계(earlySingletonObjects) 확인 → 없음
9. 3단계(singletonFactories) 확인 → A의 Factory 있음!
10. Factory로 A의 early reference 꺼내서 2단계로 승격
11. B에 A의 껍데기 주입 완료 → B 초기화 완료 → 1단계로 승격
12. A에 B 주입 완료 → A 초기화 완료 → 1단계로 승격
```
