# 정규 표현식

## 1. 정규 표현식 정리

### 정규식 기본 문법 기호

#### .

> 임의의 문자 1개

- 예시
  - `a.b`: "a + 모든 문자 1개 + b"
- 단, `[.]`과 같이 대괄호 안에 `.`이 쓰인다면, `. `그자체를 의미함

<br>

#### ^

> ^다음으로 오는 문자로 시작하는 표현식을 의미함

- 예시
  - `^a`: "a"로 시작하는 단어
- 단, `[^a]`과 같이 대괄호 안에 `^`이 쓰인다면, 시작의 의미가 아니라 `아님(Not)` 또는 제외를 의미함
  - `[^a]`: "a가 아닌 문자 1개"

<br>

#### $

> $앞의 문자로 표현식이 끝남을 의미

- 예시
  - `a$`: "a로 끝나는 단어"

<br>

#### [ ]

> [ ] 안에 해당하는 문자가 있는지를 확인 => [ ] 안의 문자 중 한 개의 문자와 매칭하는가를 확인

- 예시
  - `[abc]`: a, b, c 중 하나의 문자와 일치하는 단어
    - "a"는 정규식과 일치하는 문자인 "a"가 있으므로 매치됨
    - "before"는 정규식과 일치하는 문자인 "b"가 있으므로 매치됨

#### -

> `-` 사이의 문자 혹은 숫자와 일치

- 예시
  - `[a-z]`: "알파벳 소문자 a부터 z까지 하나와 일치"
  - `[a-z0-9]`: "알파벳 소문자 전체와 0 ~ 9 중 하나와 일치"

#### |

> 또는(OR)의 의미

- 예시
  - `A|B`: "A 혹은 B와 일치"

<br>

#### ()

> ()안의 표현식을 그룹화

- 예시
  - `01(0|1|[6 - 9])`: "01 다음에 0 또는 1 또는 6 ~ 9가 하나 있는 문자와 일치"
    - "010"은 "01" 다음에 "0" 문자가 왔으므로 정규식과 일치
    - "016"은 "01" 다음에 "6" 문자가 왔으므로 정규식과 일치
    - "012"는 "01" 다음에 "2" 문자가 왔으므로 불일치

<br>

#### 자주 쓰이는 패턴

| 패턴 | 의미                                                       |
| ---- | ---------------------------------------------------------- |
| `\b` | 구분자를 의미(공백, 탭, `,`, `/` 등)                       |
| `\B` | `\b`의 부정, 공백이나 탭이 아닌 문자인 경우에만 매칭       |
| `\d` | 0 ~ 9 사이의 숫자(`[0-9]`와 동일)                          |
| `\D` | `\d`의 부정, 숫자가 아닌 어떤 문자를 의미(`[^0-9]`와 동일) |
| `\s` | 공백, 탭                                                   |
| `\S` | 공백, 탭이 아닌 문자                                       |
| `\w` | 알파벳 대소문자, 숫자, _(언더바) (`[a-zA-Z_0-9]`와 동일)   |
| `\W` | `\w`의 부정, `[^a-zA-Z_0-9]`와 동일                        |

<br>

### 정규식 수량 기호

#### {}

> {} 안에 오는 개수만큼 온다

- 예시
  - `a{3}b`: "a"가 3번 온 다음에 b가 온다
    - "aaab"는 "a"가 3번 온 다음에 "b"가 왔으므로 일치
    - "aaaab"는 맨 처음의 "a" 다음 "a"가 3번 오고 그 다음 "b"가 왔으므로 일치
    - "aab"는 불일치

#### {n}

> 정확히 n개

#### {n, m}

> n개 이상 m개 이하

#### {n,}

> n개 이상

<br>

#### ?

> 앞의 표현식이 최대 한개까지만 온다(없을 수도 있음)

- 예시
  - `a1?` : "1"이 최대 한 개 올 수도 있고 안 올 수도 있음

#### *

> 앞의 표현식이 0개 이상 온다

- 예시
  - `a1*`: "1"이 여러 개 오거나 아예 안 올 수도 있음

#### +

> 앞의 표현식이 1개 이상 온다

- 예시
  - `a1+` : "1"이 한 개 이상 나옴

<br>

### 정규식 그룹, 캡처 기호

#### 캡처?

정규식에서 캡처(capturing)란 `일치하는 부분 문자열을 그룹화하고 추출하는 과정`을 의미한다.

캡처와 비캡처가 있는데, 캡처 그룹을 사용하게 되면 정규식 매칭 결과에서 해당 그룹에 일치하는 부분 문자열에 따로 접근할 수 있다.

#### ()

> 캡처 그룹

- 예시
  - `(foo)bar`: "foobar"와 매칭되며, "foo"는 캡처 그룹으로 지정되어 해당 부분 문자열을 추출 가능

#### (?:)

> 비캡처 그룹

- 예시
  - `(?:foo|bar)+`: "foo"나 "bar"라는 문자열이 하나 이상 나오는 문자열과 매칭됨. 단, 비캡처 그룹이므로 해당 문자열들과 패턴이 일치하는지만 중요하고 따로 "foo"나 "bar"와 같은 부분 문자열에 접근할 수는 없다.

#### (?=)

> 전방 탐색(Positive Lookahead), 일치하는 패턴을 전방에서 찾음

`(?=...)`형태로 사용되며 `...`에 일치할 패턴이 들어감

- 예시
  - `foo(?=bar)`: "foo"를 찾으며 "foo" 뒤에 "bar"가 있는 경우에만 일치

#### (?<=)

> 후방 탐색(Positive Lookbehind), 일치하는 패턴을 후방에서 찾음

`(?<=...)`형태로 사용되며 `...`에 일치할 패턴이 들어감

- 예시
  - `(?<=foo)bar`: "bar"를 찾으며, "foo"가 "bar" 앞에 있는 경우에만 일치

<br>

### 자주 사용되는 정규식 샘플 예제

- `^[0-9]*$` : 숫자(Only)
- `^[a-zA-Z]*$` : 영문자(Only)
- `[가-힣]*$` : 한글(Only)
- `\\w+@\\w+\\.\\w+(\\.\\w+)?` : e-mail 패턴(역슬래시(`\`)를 두 번 사용한 이유는 프로그래밍 언어 때문임)
